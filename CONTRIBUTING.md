# Contributing to Turno

Thanks for your interest in Turno.

## Workflow

- All changes must go through Pull Requests
- Do not commit directly to `main`
- Create a feature branch for any work

## Branch naming

- feature/feature-name
- fix/bug-name
- docs/something
- refactor/module-name

## Commits

Use simple, clear messages:

- feat: add home screen layout
- fix: correct matchmaking logic
- docs: update README

## Pull Requests

Each PR should include:

- what was done
- why it was done
- how to test it

## Code Style

- keep code simple and readable
- avoid overengineering
- reuse components when possible

## Goal

We aim to build a clean, scalable, and maintainable product.
